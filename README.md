# Promotional Codes API

Тестовый Django REST API проект для работы с:

- товарами,
- категориями,
- заказами,
- промокодами.

Проект позволяет создавать заказы с товарами и применять промокоды с учётом ограничений:
- срок действия промокода,
- максимальное количество использований,
- запрет повторного использования одним пользователем,
- ограничение по категориям товаров,
- исключение отдельных товаров из промо-акций.

---

## Стек технологий

- Python 3.12
- Django 4.2
- Django REST Framework
- drf-yasg
- Poetry
- SQLite

---

## Структура проекта

- `users` — кастомная модель пользователя
- `goods` — категории и товары
- `orders` — заказы, позиции заказа, промокоды
- `orders/management/commands/fill_db.py` — команда заполнения БД тестовыми данными

# Установка и запуск проекта

## 1. Клонировать проект

```bash
git clone <URL_репозитория>
cd promocodes
```

---

## 2. Установить Poetry

Если Poetry ещё не установлен:

```bash
pip install poetry
```

Проверить:

```bash
poetry --version
```

---

## 3. Создать виртуальное окружение через Poetry

В корне проекта выполнить:

```bash
poetry install
```

Эта команда:
- создаст виртуальное окружение,
- установит все зависимости из `pyproject.toml`.

Если нужно явно активировать shell Poetry:

```bash
poetry shell
```

Если `poetry shell` не используется, команды можно запускать так:

```bash
poetry run python manage.py runserver
```

---

## 4. Создать `.env` файл

В корне проекта рядом с `manage.py` необходимо создать файл `.env`.

Пример содержимого:

```env
DB_NAME=codek_db
DB_USER=codek
DB_PASSWORD=123456789abcdef
DATABASE_NAME=db.sqlite3
SECRET_KEY=django-insecure-my-super-secret-key-123
DEBUG=True
ALLOWED_HOSTS=127.0.0.1,localhost
```

> В текущем проекте используется SQLite, поэтому реально используется поле:
>
> - `DATABASE_NAME=db.sqlite3`
>
> Остальные переменные (`DB_NAME`, `DB_USER`, `DB_PASSWORD`) сейчас не задействованы, но могут быть полезны при переходе на PostgreSQL.

---

## 5. Применить миграции

```bash
poetry run python manage.py migrate
```

## 6. Заполнить БД тестовыми данными

## 7. Запустить проект

```bash
poetry run python manage.py runserver
```


# API Endpoints

## 1. Создание заказа

### Endpoint

```http
POST /api/orders/
```

### Полный URL

```text
http://127.0.0.1:8000/api/orders/
```

### Назначение

Создаёт новый заказ для пользователя.

При создании заказа:
- проверяется существование пользователя;
- проверяется существование товаров;
- проверяется корректность количества;
- при наличии промокода проверяется его валидность;
- создаются позиции заказа;
- рассчитываются:
  - общая стоимость без скидки;
  - сумма скидки;
  - итоговая стоимость заказа;
- увеличивается счётчик использования промокода.

---

### Пример запроса

```json
{
    "user_id": 3,
    "goods": [
        {"good_id": 1, "quantity": 2},
        {"good_id": 3, "quantity": 1}
    ],
    "promo_code": "SALE10"
}
```

---

### Описание полей

- `user_id` — ID пользователя, который оформляет заказ
- `goods` — список товаров
  - `good_id` — ID товара
  - `quantity` — количество товара
- `promo_code` — строка промокода, необязательное поле

---

### Возможные проверки

При создании заказа API проверяет:

- существует ли пользователь;
- существует ли товар;
- не пустой ли список товаров;
- положительное ли количество товаров;
- существует ли промокод;
- не истёк ли промокод;
- не достигнут ли лимит использования промокода;
- не использовал ли пользователь уже этот промокод;
- подходит ли промокод под категории товаров;
- не участвует ли товар в акции.

---

### Пример успешного ответа

```json
{
    "items": [
        {
            "good_id": 1,
            "quantity": 2,
            "price": "50000.00",
            "discount": "0.10",
            "total": "90000.00"
        },
        {
            "good_id": 3,
            "quantity": 1,
            "price": "1000.00",
            "discount": "0.10",
            "total": "900.00"
        }
    ],
    "price": "101000.00",
    "discount": "10100.00",
    "total": "90900.00"
}
```

> Обрати внимание: если товар не участвует в акции или не подходит по категории, API вернёт ошибку валидации.

---

### Пример ошибки

```json
{
    "promo_code": [
        "Промокод просрочен"
    ]
}
```

или

```json
{
    "goods": [
        "Товар с id 999 не найден"
    ]
}
```

или

```json
{
    "user_id": [
        "Пользователь не найден"
    ]
}
```

# Примечания

## 1. `.env`
Файл `.env` обязателен, так как из него читаются:
- `SECRET_KEY`
- `DEBUG`
- `ALLOWED_HOSTS`
- `DATABASE_NAME`

---

## 2. База данных
Сейчас проект использует SQLite:

```python
DATABASES = {
    "default": {
        "ENGINE": "django.db.backends.sqlite3",
        "NAME": os.getenv("DATABASE_NAME", BASE_DIR / "db.sqlite3"),
    }
}
```

Поэтому достаточно указать:

```env
DATABASE_NAME=db.sqlite3
```

---

## 3. Повторный запуск `fill_db`
Команда заполнения БД старается не дублировать часть данных при повторном запуске, но лучше использовать её на чистой или тестовой базе.
