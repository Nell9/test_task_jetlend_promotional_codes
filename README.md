Промокоды при создании заказа
В системе есть пользователи, заказы, товары и промокоды. При создании заказа пользователь может указать промокод. Промокод даёт процентную скидку на заказ с некоторыми условиями.
Необходимо реализовать HTTP endpoint для создания заказа (описание тела запроса и ответа указаны ниже). Проработайте необходимые модели данных, валидацию входящих данных, работу с базой данных.
Правила применения промокода:
Промокод должен существовать;
Промокод не должен быть просрочен;
У промокода есть ограничение на максимальное количество использований;
Один пользователь не может использовать один и тот же промокод более одного раза;
Промокод может быть ограничен определенной категорией товаров;
Некоторые товары могут быть исключены из любых акций, к ним промокод не должен применяться.

Требования к решению
Python 3.10+
Django 4.2+
решение должно быть оформлено как Django-проект;
база данных — SQLite или PostgreSQL;
проект должен запускаться локально.

// Пример тела запроса
{
  "user_id": 1,
  "goods": [ { "good_id": 1, "quantity": 2 } ],
   // Поле promo_code может отсутствовать.
  "promo_code": "SUMMER2025"
}
// Пример тела ответа
{
    "user_id": 1,
    "order_id": 1,
    "goods": [ { "good_id": 1, "quantity": 2, "price": 100, "discount": "0.1", "total": 180 } ],
    "price": 200,
    "discount": "0.1",
    "total": 180,
}
